# 게임 개선 수정사항

## 1. 판정 윈도우 조정 (우선순위: 높음)

**파일**: `src/config/judgeConfig.ts`

**문제**: 판정 윈도우가 너무 넓어 게임이 쉬워 보임
- Perfect: ±80ms (일반 리듬게임은 ±40~50ms)
- Great: ±160ms
- Good: ±240ms

**수정 요청**:
- Perfect를 ±40~50ms로 축소
- Great를 ±80~100ms로 축소
- Good를 ±120~150ms로 축소
- 롱노트 판정은 일반 판정의 1.2~1.3배로 조정 (현재 1.5배)

---

## 2. 게임 루프 업데이트 간격 개선 (우선순위: 중간)

**파일**: `src/hooks/useGameLoop.ts` (87번째 줄)

**문제**: `TIME_UPDATE_INTERVAL_MS = 50ms`로 인한 반응 지연감

**수정 요청**:
- 업데이트 간격을 16ms (60fps 기준) 또는 33ms (30fps 기준)로 축소
- 또는 requestAnimationFrame 기반으로 매 프레임 업데이트하되, 상태 업데이트는 최적화

---

## 3. 키 입력 자동 해제 로직 개선 (우선순위: 중간)

**파일**: `src/hooks/useGameJudging.ts` (138-146번째 줄)

**문제**: 키 입력 후 100ms 자동 해제가 부자연스러움

**수정 요청**:
- 실제 키 릴리스 이벤트에만 반응하도록 변경
- 자동 해제 타이머 제거 또는 지연 시간 조정 (50ms 이하)

---

## 4. YouTube 오디오 동기화 개선 (우선순위: 높음)

**파일**: `src/hooks/useTestYoutubePlayer.ts`, `src/hooks/useTestAudioController.ts`

**문제**: 
- 3초 오프셋이 하드코딩되어 있음
- 동기화 불안정 가능성

**수정 요청**:
- 오프셋을 설정 가능하게 만들거나 자동 보정 로직 추가
- 오디오와 게임 시간 동기화 검증 및 보정 메커니즘 구현

---

## 5. 에러 처리 강화 (우선순위: 높음)

**파일**: `src/components/Game.tsx`, `src/components/ChartSelect.tsx`

**문제**: 
- YouTube API 실패 시 명확한 에러 메시지 부족
- 채보 로드 실패 시 크래시 가능성

**수정 요청**:
- YouTube API 로드 실패 시 사용자 친화적 에러 메시지 표시
- 채보 로드 실패 시 에러 토스트/모달 표시 및 복구 옵션 제공
- try-catch로 에러 핸들링 강화

---

## 6. 초기 사용자 가이드 추가 (우선순위: 낮음)

**파일**: `src/components/GameMenu.tsx` 또는 새 컴포넌트

**문제**: 첫 플레이 시 조작법을 알기 어려움

**수정 요청**:
- 첫 방문 시 간단한 튜토리얼 오버레이 추가
- 키 조작법, 판정 시스템 설명
- "다시 보지 않기" 옵션 제공

---

## 7. UI/UX 개선 (우선순위: 낮음)

**파일**: `src/components/GameMenu.tsx`, `src/components/ChartSelect.tsx`

**문제**: 
- 메인 메뉴가 단순함
- 채보 선택 화면이 밋밋함

**수정 요청**:
- 메인 메뉴에 애니메이션/시각 효과 추가
- 채보 선택 화면에 미리보기, 난이도 표시 등 정보 강화
- 로딩 상태 표시 개선

---

## 참고사항

- 모든 수정은 기존 기능을 깨뜨리지 않도록 주의
- 판정 윈도우 변경 시 기존 채보와의 호환성 고려
- 성능 최적화 시 렌더링 프레임레이트 유지 확인
